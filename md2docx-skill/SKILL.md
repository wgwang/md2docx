# md2docx Agent Skill

This skill allows AI agents (like OpenClaw, AutoGPT, or Gemini CLI itself) to convert Markdown strings into professionally formatted DOCX files.

## Workflow

1. **Input:** A Markdown string containing the document content.
2. **Action:** Call the `md2docx` Python API or CLI.
3. **Output:** A `.docx` file or a byte stream for further transmission.

## Using the Python API

For in-memory processing (recommended for remote agents), use the `convert_to_bytes()` method.

```python
from md2docx import MarkdownToDocx
import io

md_content = "# Hello World\nGenerated by AI Agent."
converter = MarkdownToDocx(template_path="professional_template.docx")
docx_stream = converter.convert_to_bytes(md_content)

# Use docx_stream (io.BytesIO) for uploads, emails, or saving
with open("agent_output.docx", "wb") as f:
    f.write(docx_stream.getbuffer())
```

## Using the CLI

For local file generation:

```bash
md2docx input.md output.docx --template professional_template.docx
```

## Template Styles

Available templates in this repository:
- `professional_template.docx`: Business and professional style.
- `academic_template.docx`: Rigorous academic style (SimSun/Times New Roman).
- `creative_template.docx`: Colorful and artistic style.

## Key Features for Agents

- **Math Support:** LaTeX math is automatically converted to native Word equations.
- **Image Handling:** Remote URLs in Markdown are automatically fetched and embedded.
- **CJK Font Support:** Fixes common issues with Chinese/Japanese/Korean fonts in DOCX.

## Bundled Resources

- **Helper Script:** `scripts/md2docx_helper.py` - A standalone CLI tool for agents to convert stdin (markdown) to stdout (docx bytes) or files.
